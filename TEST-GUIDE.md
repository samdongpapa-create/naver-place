# 🧪 테스트 가이드

## 입력하신 URL 테스트

**입력 URL:**
```
https://map.naver.com/p/entry/place/1443688242
```

**자동 변환:**
```
https://m.place.naver.com/place/1443688242
```

## 실행 방법

```bash
# 1. 프로젝트 디렉토리로 이동
cd naver-place-optimizer

# 2. 의존성 설치
npm install

# 3. Playwright 브라우저 설치
npx playwright install chromium --with-deps

# 4. 서버 실행
npm run dev
```

## 테스트 절차

1. 브라우저에서 `http://localhost:3000` 접속

2. URL 입력창에 붙여넣기:
   ```
   https://map.naver.com/p/entry/place/1443688242
   ```

3. "진단하기" 버튼 클릭

4. 로딩 중... (10-30초 소요)

5. 결과 확인:
   - ✅ 플레이스 이름
   - ✅ 주소
   - ✅ 종합 점수
   - ✅ 5개 항목별 점수
     - 상세설명
     - 오시는길
     - 대표키워드
     - 리뷰
     - 사진

## 예상 결과

### 성공 시
- 플레이스 정보가 정확히 표시됨
- 5개 항목 모두 점수 표시
- 각 항목별 개선점 표시

### 실패 가능한 경우

**1. iframe 타임아웃**
```
페이지 로딩에 실패했습니다. URL을 다시 확인해주세요.
```
→ 해결: 다시 시도하거나 Railway 배포 후 테스트

**2. 일부 정보 누락**
- 상세설명/오시는길이 없는 경우: 해당 항목 0점
- 리뷰/사진이 없는 경우: 해당 항목 0점
- 키워드가 없는 경우: 해당 항목 0점
→ 이는 정상입니다. 실제로 정보가 없는 것입니다.

## 유료 진단 테스트

1. 무료 진단 완료 후
2. "유료 컨설팅 받기 (₩19,900)" 버튼 클릭
3. 모달 팝업에서 검색어 입력:
   ```
   [해당 업체가 속한 지역 + 업종]
   예: 강남 카페, 이태원 맛집
   ```
4. "결제하고 진단받기" 클릭
5. 경쟁사 5개 분석 결과 확인

## 문제 발생 시

### 크롤링 실패
1. `npm run dev`로 서버 재시작
2. 브라우저 새로고침
3. 다른 플레이스 URL로 테스트

### 메모리 부족 (Railway)
- Free Tier: 512MB RAM (부족할 수 있음)
- 해결: Pro 플랜 업그레이드 ($5/월)

### 네이버 차단
- IP 차단 가능성
- 해결: Railway 재배포 (IP 변경됨)
